## Method

AI products fail in two ways: **the model is wrong** (caught by eval rubrics + calibration) or **the model is right but the question was framed wrong** (caught by verifier-gated retrieval). Most AI-PM work optimises for one and ignores the other.

My signature methodology — **5-dim evaluation rubric + verifier-gated retrieval** — recurs across every project here. It started at DOPEDCLUB (Casablanca), got formalised at Baidu ERNIE (Shenzhen), and now runs in production at jak.ma (Morocco).

The five dimensions:

1. **pricing taxonomy** — what to charge for, what to bundle
2. **evaluation rubric** — what "good" actually means for the model
3. **calibration protocol** — how to keep that rubric stable across runs
4. **verifier philosophy** — what to enforce in code vs trust the model with
5. **multimodal decision** — when to escalate to vision, audio, or human
